# 工程地基补全清单（P0：第 1-3 月）

> 目标：四大能力栈各自跑通一个最小 demo。不求精通，求"能独立从零搭起来"。
> 学习顺序：LLM 应用栈 → KG 工程栈 → PyTorch 训练栈 → 图学习栈 → 部署栈（边做主项目边补）。

每个栈给出：要会什么、最快上手路径、验收 demo、推荐资源。

---

## 栈 1：LLM 应用栈（最先学，直接服务主项目）

### 要会什么
- LLM API 调用：多轮对话、流式输出、错误重试、token 统计
- Tool Use / Function Calling：让模型调用外部工具
- RAG 框架：LlamaIndex 或 LangChain，能跑通"文档 → 切块 → 向量化 → 检索 → 生成"
- 向量数据库：Chroma（本地最简）/ Qdrant，理解 embedding + 相似度检索
- Agent 编排：LangGraph 状态机，理解 ReAct 循环

### 最快上手路径
1. 直接复用已有脚手架 `资源/第二阶段-AI工程/claude-api-starter/`，跑通多轮/流式/Tool Use 三个示例。
2. 用 LlamaIndex 官方 5 行代码 quickstart 跑通最小 RAG，喂自己的几篇 PDF。
3. 用 LangGraph 实现一个最简 ReAct Agent（一个搜索工具 + 一个计算工具）。

### 验收 demo
用 LlamaIndex + 本地 Chroma 跑通一个文档问答：上传 3 篇 PDF，能正确回答其中的事实问题，并返回引用来源。

### 推荐资源
- LlamaIndex 文档：https://developers.llamaindex.ai/
- LangChain 文档：https://docs.langchain.com/
- LangGraph：https://docs.langchain.com/oss/python/langgraph/overview
- 已有：`资源/第一阶段-开发工作流/AI辅助开发工作流.md`

## 栈 2：KG 工程栈（主项目的核心差异化能力）

### 要会什么
- Neo4j 安装与使用，Cypher 查询语言（CREATE / MATCH / 路径查询）
- 图数据建模：节点、关系、属性的设计
- 实体抽取（NER）：GLiNER（零样本）
- 关系抽取（RE）：GLiREL（零样本）/ REBEL（端到端三元组）
- 文档 → 三元组 → 入图的完整流水线

### 最快上手路径
1. Neo4j Desktop 或 Docker 起一个本地实例，跟 GraphAcademy 免费课走一遍 Cypher 基础。
2. 用 GLiNER 对一篇文档抽实体，用 REBEL 抽关系三元组。
3. 把三元组写进 Neo4j，用 Cypher 查多跳路径（如"A 和 C 通过什么关联"）。

### 验收 demo
输入一篇文档 → 自动抽取实体和关系 → 写入 Neo4j → 能用 Cypher 查询出一条两跳推理路径。

### 推荐资源
- Neo4j GraphAcademy（免费认证）：https://graphacademy.neo4j.com/
- GLiNER：https://github.com/urchade/GLiNER
- GLiREL：https://github.com/jackboyla/GLiREL
- REBEL：https://github.com/Babelscape/rebel
- KG+LLM Roadmap 综述：arXiv:2306.08302

## 栈 3：PyTorch 训练栈（副项目与微调必需）

### 要会什么
- Dataset / DataLoader 自定义
- 训练循环：前向、loss、反向、优化器、学习率调度
- 混合精度（amp）、梯度累积、断点续训
- 多卡基础（DDP 概念，单机多卡能跑起来）
- 对比学习 loss（InfoNCE，副项目要用）
- 微调技术：LoRA / QLoRA（PEFT 库）

### 最快上手路径
1. 手写一个完整训练循环：在 CIFAR-10 上训一个小 CNN，含 checkpoint 保存/恢复。
2. 用 HuggingFace Trainer + PEFT 对一个小 LLM 做 LoRA 微调。
3. 理解 CLIP 的对比 loss，为副项目复现 Structure-CLIP 打基础。

### 验收 demo
在一个小数据集上，用 LoRA 微调一个模型，保存权重，加载后推理验证效果有提升。

### 推荐资源
- PyTorch 官方教程：https://pytorch.org/tutorials/
- HuggingFace PEFT 文档（LoRA/QLoRA）
- open_clip（对比训练参考）：https://github.com/mlfoundations/open_clip

## 栈 4：图学习栈（结构化建模的底层能力）

### 要会什么
- PyG（PyTorch Geometric）或 DGL 的数据结构
- 经典模型：GraphSAGE、GAT、R-GCN（关系图，KG 场景重要）
- 任务：节点分类、链接预测（KG 补全的基础）
- 图采样、mini-batch 训练

### 最快上手路径
1. 装 PyG，跑官方 Cora 节点分类示例（GCN/GAT）。
2. 在一个 KG 数据集上做链接预测（R-GCN）。
3. 理解图的消息传递机制，为主项目"图结构指导检索"做铺垫。

### 验收 demo
在标准图数据集上跑通节点分类（准确率达基线）+ 链接预测（AUC 达基线）。

### 推荐资源
- PyG 文档与官方 examples：https://pytorch-geometric.readthedocs.io/
- DGL 文档：https://www.dgl.ai/
- CS224W（斯坦福图机器学习课）公开资料

## 栈 5：部署栈（边做主项目边补，求职加分项）

### 要会什么
- FastAPI：把模型/系统包成 REST 接口，异步处理
- Docker：写 Dockerfile、docker-compose 编排多服务（API + 向量库 + Neo4j）
- vLLM：本地高吞吐推理 LLM 的基础用法
- 基本的日志、trace、压测意识

### 最快上手路径
1. 把栈 1 的 RAG demo 用 FastAPI 包成 `/query` 接口。
2. 写 docker-compose，把 API + Chroma/Neo4j 一起拉起来。
3. 了解 vLLM 如何本地部署一个开源 LLM（如 Qwen）。

### 验收 demo
`docker-compose up` 一键拉起完整 RAG 服务（API + 向量库），curl 调用返回正确结果。

### 推荐资源
- FastAPI 文档：https://fastapi.tiangolo.com/
- vLLM 文档：https://docs.vllm.ai/
- 已有：`资源/第四阶段-系统设计/系统设计基础.md`（含 Dockerfile / docker-compose 示例）
