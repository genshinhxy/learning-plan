## 1. 论文投稿策略

### 1.1 会议地图与节奏

| 方向 | 顶会 | 投稿月份（约） | 适合阶段 |
|------|------|--------------|---------|
| NLP（RAG/Agent/KG） | ACL / EMNLP / NAACL | ACL 2月、EMNLP 6月 | 主项目 |
| ML 综合 | NeurIPS / ICLR / ICML | NeurIPS 5月、ICLR 9月 | 主/副均可 |
| CV / 多模态 | CVPR / ICCV / ECCV | CVPR 11月、ICCV 3月 | 副项目 |
| AI 综合 | AAAI / IJCAI | AAAI 8月 | 主/副均可 |

### 1.2 阶梯式投稿（降低风险）
1. **先投 workshop**：门槛低、反馈快、能拿到同行评审意见。主会的 workshop（如 ACL/NeurIPS workshop）是练手好去处。
2. **再冲主会**：用 workshop 反馈打磨后投主会。
3. **被拒不慌**：根据 review 改进，转投下一个会议（NLP/ML/CV 会议错峰，全年都有窗口）。

### 1.3 写作要点
- 结构：动机（为什么重要）→ 方法（新在哪）→ 实验（多数据集）→ 消融（每个模块的贡献）→ 误差分析。
- **消融实验是审稿人最看重的**——证明你的每个设计都有用。
- 开源代码 + 数据/评测集，大幅提升可信度和引用。
- 多模态评测类工作天然适合发表（社区缺好 benchmark）。

## 2. 简历包装

### 2.1 一句话定位
"做结构化知识增强的大模型的算法工程师"——开头就让 HR/面试官记住你的差异化。

### 2.2 两个代表作的四段式写法
每个项目用 **问题 → 方法 → 结果 → 工程** 四段：

**代表作 1：GraphRAG-Agent 系统**
- 问题：复杂多跳文档问答中，朴素 RAG 检索不足、易幻觉。
- 方法：知识图谱构建 + 三路混合检索 + 图结构指导的 Agent 多跳推理。
- 结果：多跳问题准确率较 baseline 提升 X%，引用正确率提升 Y%（投稿/在投）。
- 工程：FastAPI + Docker 部署，含 trace、缓存、压测，并发延迟 Z ms。

**代表作 2：VLM 细粒度对齐评测**
- 问题：现代 VLM 在组合性/细粒度图文对齐上仍有短板，评测易受语言偏置污染。
- 方法：基于场景图的结构化负样本 + 去偏评测基准。
- 结果：揭示主流 VLM 的 X 类弱项，构建评测集（开源）。
- 工程：评测 pipeline 自动化，覆盖 N 个主流 VLM。

### 2.3 底层能力标签（技能栏）
知识图谱（Neo4j/Cypher）、图学习（PyG/R-GCN）、RAG/检索（混合检索/rerank）、Agent（LangGraph/ReAct）、多模态（CLIP/LLaVA/Qwen-VL）、训练微调（PyTorch/LoRA）、部署（FastAPI/Docker/vLLM）、评测数据构造。

## 3. 岗位关键词与匹配

### 3.1 主线岗位（投递关键词）
- 大模型应用算法、RAG 算法、检索增强、知识图谱算法、Agent 研发、AI Search、对话系统算法。

- 多模态算法、视觉语言模型、图文理解、多模态评测。
- JD 里出现"VLM/图文对齐/多模态大模型/CLIP"→ 用代表作 2 主打。

### 3.3 投递策略
- 主线为主（70%），副线为辅（30%）。
- 优先投有明确 RAG/Agent 或多模态团队的公司（大厂的大模型团队、AI 应用创业公司）。

## 4. 面试准备

### 4.1 四个板块
1. **基础八股**：Transformer/注意力机制、RAG 三范式、向量检索原理、对比学习、LoRA、Agent（ReAct/反思）、知识图谱基础、图神经网络。
2. **项目深挖**：能讲清两个代表作的每个技术决策"为什么这么做、有没有别的方案、效果如何验证"。这是重点，面试官会刨根问底。
3. **系统设计**：如何设计一个支持百万文档的 RAG 系统？如何降低延迟/成本？如何评测和监控？（复用 `资源/第四阶段-系统设计/系统设计基础.md`）
4. **手撕代码**：LeetCode 中等难度（数组/字符串/树/图/DP），外加可能的"手写注意力/手写对比 loss"。

### 4.2 高频问题预判
- "你的 GraphRAG 相比朴素 RAG 好在哪？图结构具体怎么帮助检索？"
- "难负样本不是被证明有偏置吗，你怎么避免？"（考你读没读 Hard Positive Truth）
- "线上怎么保证延迟？怎么评测系统质量？怎么发现幻觉？"
- "知识图谱怎么自动构建？实体消歧怎么做？"

### 4.3 准备方法
- 每个代表作写一份"项目 FAQ"，预演 20 个深挖问题。
- 八股按板块整理笔记，结合论文理解而非死记。
- 手撕代码每周保持手感（不用刷太多，保持节奏即可）。

| 时间 | 动作 |
|------|------|
| 第 18-20 月 | 主项目有可演示成果后，开始投实习（日常实习/暑期实习） |
| 第 20-24 月 | 进入一段大厂算法实习，最好对口 RAG/Agent 或多模态团队 |
| 第 24-27 月 | 实习转化或积累后，准备秋招（论文最好此时已投出/接收） |

> 实习是最强的求职杠杆：一段对口实习 > 三个自建项目。优先级很高。

## 7. 关键提醒

1. **论文和工程不是二选一**——同一个项目两头吃，这是你方案的核心优势，别做成纯 demo 或纯刷分。
2. **实习优先级高于第三个项目**——两个扎实代表作 + 一段对口实习，胜过堆数量。
3. **被拒是常态**——论文按会议错峰反复投，求职广撒网，心态放平。
4. **差异化是护城河**——始终强调"结构化知识 + 大模型"的交叉定位，不要泯然于纯 RAG/纯调 API 的人群。
