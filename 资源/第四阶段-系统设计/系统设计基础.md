# 系统设计基础

## 核心理念

> Agent最终不是漂在空中的聊天框，而是接入真实业务系统的自动化层。

---

## 1. 分布式系统基础

### 1.1 核心概念

| 概念 | 说明 | AI应用场景 |
|------|------|-----------|
| **CAP定理** | 一致性、可用性、分区容错性不可兼得 | Agent状态管理 |
| **最终一致性** | 不要求实时一致，但最终会达到一致 | 异步任务队列 |
| **幂等性** | 同一操作执行多次结果相同 | 工具调用重试 |
| **熔断器** | 快速失败，防止级联故障 | 外部API调用 |
| **背压** | 生产者速度超过消费者时的流控 | 流式响应处理 |

### 1.2 常见架构模式

```
单体架构
    ↓ 拆分
微服务架构
    ↓ 演进
事件驱动架构
    ↓ 结合AI
Agent编排架构
```

## 2. 数据库设计

### 2.1 数据库选型

| 类型 | 适用场景 | AI应用示例 |
|------|----------|-----------|
| **PostgreSQL** | 结构化数据、事务 | 用户管理、订单系统 |
| **Redis** | 缓存、会话、队列 | 对话历史、任务队列 |
| **MongoDB** | 文档型、灵活schema | Agent配置、trace存储 |
| **向量数据库** | 语义搜索、RAG | 知识库检索 |
| **Elasticsearch** | 全文搜索、日志 | 日志分析、知识搜索 |

### 2.2 向量数据库选型

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| **Pinecone** | 全托管、易用 | 快速原型 |
| **Chroma** | 轻量级、本地优先 | 个人项目 |
| **Weaviate** | 功能丰富、混合搜索 | 企业应用 |
| **pgvector** | PostgreSQL扩展 | 已有PG的项目 |
| **Qdrant** | 高性能、Rust实现 | 大规模应用 |

### 2.3 数据建模示例

```sql
-- 对话系统数据模型
CREATE TABLE conversations (
    id UUID PRIMARY KEY,
    user_id UUID NOT NULL,
    agent_id UUID NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    metadata JSONB
);

CREATE TABLE messages (
    id UUID PRIMARY KEY,
    conversation_id UUID REFERENCES conversations(id),
    role VARCHAR(20) NOT NULL,  -- user/assistant/system/tool
    content TEXT NOT NULL,
    token_count INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE tool_calls (
    id UUID PRIMARY KEY,
    message_id UUID REFERENCES messages(id),
    tool_name VARCHAR(100) NOT NULL,
    input JSONB,
    output JSONB,
    status VARCHAR(20),  -- pending/success/failed
    latency_ms INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 向量存储（使用pgvector）
CREATE TABLE knowledge_chunks (
    id UUID PRIMARY KEY,
    document_id UUID,
    content TEXT,
    embedding vector(1536),  -- OpenAI embedding维度
    metadata JSONB,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops);
```

### 3.2 Agent任务队列设计

```python
from celery import Celery
from dataclasses import dataclass
from typing import Any

app = Celery('agent_tasks', broker='redis://localhost:6379')

@dataclass
class AgentTask:
    task_id: str
    agent_id: str
    input: str
    context: dict
    priority: int = 0

@app.task(bind=True, max_retries=3)
def execute_agent_task(self, task: AgentTask):
    """执行Agent任务，支持重试和错误处理。"""
    try:
        # 1. 加载Agent配置
        agent = load_agent(task.agent_id)

        # 2. 执行Agent
        result = agent.run(task.input, context=task.context)

        # 3. 保存结果
        save_result(task.task_id, result)

        # 4. 发送完成通知
        notify_completion.delay(task.task_id, result)

        return result

    except RetryableError as exc:
        # 可重试的错误
        raise self.retry(exc=exc, countdown=60)

    except Exception as exc:
        # 不可重试的错误
        save_error(task.task_id, str(exc))
        raise

@app.task
def notify_completion(task_id: str, result: Any):
    """通知任务完成。"""
    # WebSocket推送、邮件通知等
    pass
```

## 4. 权限与安全

### 4.1 认证方案

| 方案 | 适用场景 | 优缺点 |
|------|----------|--------|
| **JWT** | 无状态API | 简单，但无法即时撤销 |
| **Session** | 有状态应用 | 可控，但需要存储 |
| **OAuth 2.0** | 第三方授权 | 标准，但复杂 |
| **API Key** | 服务间调用 | 简单，但安全性较低 |

### 4.2 Agent权限设计

```python
from enum import Enum
from dataclasses import dataclass

class Permission(Enum):
    READ = "read"
    WRITE = "write"
    EXECUTE = "execute"
    ADMIN = "admin"

@dataclass
class AgentPermissions:
    """Agent权限配置。"""

    # 工具权限
    allowed_tools: list[str]  # 允许调用的工具
    denied_tools: list[str]   # 禁止调用的工具

    # 数据权限
    readable_resources: list[str]  # 可读取的资源
    writable_resources: list[str]  # 可写入的资源

    # 操作权限
    max_iterations: int       # 最大循环次数
    max_tokens: int           # 最大token消耗
    require_approval: list[str]  # 需要人工审批的操作

    # 网络权限
    allowed_domains: list[str]  # 允许访问的域名
    denied_domains: list[str]   # 禁止访问的域名

def check_permission(agent: Agent, action: str, resource: str) -> bool:
    """检查Agent是否有权限执行操作。"""

    perms = agent.permissions

    # 检查工具权限
    if action == "tool_call":
        tool_name = resource
        if tool_name in perms.denied_tools:
            return False
        if perms.allowed_tools and tool_name not in perms.allowed_tools:
            return False

    # 检查数据权限
    if action == "read":
        if resource not in perms.readable_resources:
            return False

    if action == "write":
        if resource not in perms.writable_resources:
            return False

    return True
```

## 5. 性能优化

### 5.1 缓存策略

| 策略 | 适用场景 | 实现方式 |
|------|----------|----------|
| **Prompt缓存** | 重复的系统提示 | Anthropic API cache_control |
| **响应缓存** | 相同输入相同输出 | Redis + hash(input) |
| **Embedding缓存** | 文档向量化 | 本地缓存或向量数据库 |
| **会话缓存** | 对话历史 | Redis或内存 |

### 5.2 Prompt缓存最佳实践

```python
# 正确：稳定内容在前，动态内容在后
response = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=16000,
    cache_control={"type": "ephemeral"},
    system=[
        {
            "type": "text",

            "cache_control": {"type": "ephemeral"}
        },
        {
            "type": "text",
            "text": dynamic_context  # 每次可能不同的上下文
            # 不加cache_control
        }
```

### 5.3 并发处理

```python
import asyncio
from anthropic import AsyncAnthropic

client = AsyncAnthropic()

async def process_batch(messages: list[str]) -> list[str]:
    """并发处理多个消息。"""

    async def single_call(msg: str) -> str:
        response = await client.messages.create(
            model="claude-opus-4-7",
            max_tokens=16000,
            messages=[{"role": "user", "content": msg}]
        )
        return response.content[0].text

    # 并发执行，限制并发数
    semaphore = asyncio.Semaphore(10)

    async def limited_call(msg: str) -> str:
        async with semaphore:
            return await single_call(msg)

    tasks = [limited_call(msg) for msg in messages]
    return await asyncio.gather(*tasks)
```

## 6. 容器化部署

### 6.1 Dockerfile示例

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY . .

# 设置环境变量
ENV PYTHONUNBUFFERED=1
ENV ANTHROPIC_API_KEY=""

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```
