# AI辅助开发工作流

## 核心理念

> 未来强的人不是"会prompt"，而是能把问题表达成AI可执行的工程结构。

---

## 1. Prompt Engineering 工程化

### 1.1 结构化Prompt模板

```markdown
# 角色
你是一个[具体角色]，擅长[具体技能]。

# 任务
[清晰描述要完成的任务]

# 输入
[描述输入数据的格式和内容]

# 输出要求
- 格式: [JSON/Markdown/代码等]
- 长度: [具体要求]
- 语言: [中文/英文]

# 约束
- [不要做什么]
- [必须包含什么]

# 示例（可选）
输入: ...
输出: ...
```

### 1.2 常用Prompt模式

| 模式 | 适用场景 | 示例 |
|------|----------|------|
| **Chain of Thought** | 复杂推理 | "让我们一步一步思考" |
| **Few-shot** | 格式转换 | 提供2-3个输入输出示例 |
| **Role Playing** | 专业任务 | "你是一个资深后端工程师" |
| **Structured Output** | 数据提取 | "以JSON格式输出，包含以下字段" |
| **Self-Critique** | 质量提升 | "检查你的回答是否有错误" |

### 1.3 Prompt版本管理

```
prompts/
├── v1.0/
│   ├── classify.txt
│   ├── extract.txt
│   └── generate.txt
├── v1.1/
│   ├── classify.txt  # 修复了边界case
│   └── ...
└── current -> v1.1   # 软链接指向当前版本
```

## 2. AI辅助代码阅读

### 2.1 快速理解代码库

```
提示词模板：
"请帮我理解这段代码的作用：
1. 它的主要功能是什么？
2. 输入和输出分别是什么？
3. 关键的数据流是怎样的？
4. 有哪些潜在的问题或改进点？

代码：
[粘贴代码]"
```

### 2.2 代码审查助手

```
提示词模板：
"请审查以下代码，重点关注：
1. 安全漏洞
2. 性能问题
3. 代码风格
4. 边界条件处理
5. 错误处理

请以优先级排序的问题列表形式输出。"
```

### 2.3 重构建议

```
提示词模板：
"这段代码需要重构，目标是[具体目标]。
请提供：
1. 重构方案（保持功能不变）
2. 具体的代码修改建议
3. 需要添加的测试用例"
```

## 3. AI辅助测试生成

### 3.1 单元测试生成

```python
def generate_tests(function_code: str, framework: str = "pytest") -> str:
    """让AI生成单元测试。"""

    prompt = f"""请为以下函数生成{framework}单元测试：

```python
{function_code}
```

要求：
1. 覆盖正常路径
2. 覆盖边界条件
3. 覆盖异常输入
4. 使用pytest的fixture和parametrize
5. 添加中文注释说明每个测试的目的
"""

    return call_claude(prompt)
```

### 3.2 测试用例设计

```
提示词模板：
"函数签名: def process_order(order_id: str, items: list) -> dict
请设计测试用例，包括：
1. 正常场景（2-3个）
2. 边界条件（空列表、单个item、最大数量）
3. 异常场景（无效order_id、items格式错误）
4. 并发场景（如有意义）

每个用例包含：输入、预期输出、测试目的"
```

## 4. AI辅助需求拆解

### 4.1 需求拆解模板

```
原始需求：[用户给出的模糊需求]

请帮我拆解为可执行的工程任务：

1. 功能点列表
   - [ ] 核心功能1：描述
   - [ ] 核心功能2：描述
   - [ ] 辅助功能1：描述

2. 技术实现
   - 需要的API/库
   - 数据结构设计
   - 接口定义

3. 验收标准
   - 功能验收：具体可测试的条件
   - 性能验收：延迟、吞吐量等指标

4. 风险点
   - 技术风险
   - 依赖风险
   - 时间风险
```

### 4.2 架构设计辅助

```
提示词模板：
"我需要设计一个[系统名称]，主要功能是[功能描述]。
请帮我：
1. 画出系统架构图（用文字描述）
2. 列出核心模块和它们的职责
3. 定义模块间的接口
4. 识别潜在的扩展点"
```

## 6. Git工作流与AI

### 6.1 Commit Message生成

```bash
# 生成commit message
git diff --staged | claude -p "请为这些改动生成一个简洁的commit message，格式为：
<type>(<scope>): <subject>

其中type可以是feat/fix/refactor/docs/test/chore"
```

### 6.2 PR描述生成

```bash
# 生成PR描述
git log main..HEAD --oneline | claude -p "请根据这些commit生成PR描述，包含：
1. 变更摘要（3-5个要点）
2. 测试计划
3. 注意事项"
```

## 7. 实践练习

### 练习1：AI辅助重构
1. 找一个你之前写的项目（100-500行）
2. 用AI分析代码问题
3. 让AI生成重构方案
4. 执行重构并验证
5. 写一篇复盘文章

### 练习2：AI辅助测试
1. 选择一个没有测试的函数
2. 让AI生成测试用例
3. 运行测试，补充遗漏的case
4. 确保覆盖率 > 80%

### 练习3：AI辅助需求拆解
1. 找一个模糊的产品需求
2. 用AI拆解为可执行任务
3. 评估任务复杂度
4. 制定开发计划
