# 算法工程师成长计划（结构化知识增强大模型方向）

> 长线规划：2-3 年（在读期间），论文产出与就业能力并重。
> 主线 = LLM Agent / RAG / GraphRAG / AI Search 算法工程师
> 副线 = 多模态 / VLM 算法工程师（细粒度图文对齐 + 评测）
> 能力底座 = 知识图谱 + 图学习 + 结构化语义建模 + 评测数据构造

---

## 0. 定位与战略（先读这一节）

### 0.1 你的核心标签
把自己包装成一句话："做**结构化知识增强的大模型**的算法工程师"。
- 结构化 = 知识图谱、场景图、三元组、图结构
- 知识增强 = GraphRAG、KG+LLM、检索增强
- 大模型 = LLM Agent（主）+ VLM（副）

这个标签的稀缺性在于：纯做 RAG 的人多，纯做 KG 的人多，但**能把图结构/知识图谱真正接进大模型推理链路**的人少。你已有 Structure-CLIP、场景图、三元组抽取、难负样本的背景，正好卡在这个交叉点。

### 0.2 主线 vs 副线的取舍
- **主线（70% 精力）**：GraphRAG-Agent 系统。这是你简历的"代表作"，也是冲会议论文的主战场。
- **副线（30% 精力）**：VLM 细粒度对齐 + 评测。复用你 Structure-CLIP 的积累，升级到现代 VLM，作为"第二增长曲线"和多模态岗位的敲门砖。
- 两条线共享底座（结构化语义、评测数据构造），不是两个割裂的方向。

### 0.3 论文 + 就业并重意味着什么
- 主项目必须**同时**满足：有研究新意（能投会议）+ 工程闭环（能写进简历、能演示）。
- 评判标准：一个项目做完，既能产出一篇 workshop/会议投稿，又能在面试时画出完整系统架构图并回答"线上怎么部署、怎么压测、怎么评测"。
- 反面教材：只跑通 demo 不做消融（无法成文）；只堆 SOTA 数字不做工程（面试答不上系统问题）。

## 2. P0：工程地基（第 1-3 月）

详见 `资源/01-工程地基/工程地基补全清单.md`。四个能力栈，每个栈做到"能独立跑通一个最小 demo"即可，不求精通，后续在项目里加深。

| 能力栈 | 要会什么 | 验收 demo |
|--------|---------|----------|
| **PyTorch 训练栈** | Dataset/DataLoader、训练循环、混合精度、断点续训、多卡基础 | 在一个小数据集上微调一个分类/对比模型并保存权重 |
| **LLM 应用栈** | API 调用、流式、Tool Use、LangGraph/LlamaIndex、向量库 | 用 LlamaIndex 跑通一个最小 RAG（已有 claude-api-starter 可复用） |
| **KG 工程栈** | Neo4j + Cypher、实体/关系抽取、三元组入库 | 把一篇文档抽成三元组写进 Neo4j 并用 Cypher 查询 |
| **图学习栈** | PyG/DGL、GraphSAGE/GAT、链接预测 | 在一个标准图数据集上跑通节点分类 + 链接预测 |
| **部署栈** | FastAPI、Docker、vLLM 基础 | 把上面任一 demo 用 FastAPI 包成接口并 Docker 化 |

**学习顺序建议**：LLM 应用栈（最快出成就感、直接服务主项目）→ KG 工程栈 → PyTorch 训练栈 → 图学习栈 → 部署栈（边做主项目边补）。

## 3. P1-P2：GraphRAG-Agent 主项目（第 4-14 月）

详见 `资源/02-GraphRAG-Agent主项目/主项目路线图.md`。

**项目名**：面向复杂文档问答的 GraphRAG-Agent 系统

**七步推进**：
1. **选题与基线**（第 4 月）：跑通 LightRAG / Microsoft GraphRAG 官方 demo，建立 baseline。
2. **KG 构建**（第 5 月）：文档 → 实体/关系抽取 → 知识图谱（Neo4j），用 GLiNER/REBEL 等。
3. **混合检索**（第 6 月）：向量检索 + 图检索 + BM25，加 reranker（bge-reranker / ColBERT）。
4. **Agent 推理**（第 7 月）：ReAct/Reflexion 框架，多跳推理走图结构，用 LangGraph 编排。
5. **评测体系**（第 8 月）：构造评测集，硬指标 + LLM-as-Judge，对比 baseline。
6. **研究新意 + 消融**（第 9-12 月）：找到一个可发论文的创新点（如图结构如何指导检索/推理），做消融实验。
7. **投稿 + 工程化**（第 13-14 月）：成文投稿；同时做部署、压测、trace，写进简历。

**研究创新点候选方向**（择一深挖）：
- 图结构信号如何指导 Agent 的多跳检索路径（vs 盲目 BFS）
- KG 与文本检索结果的融合/冲突消解机制
- 用图的拓扑特征做检索结果的可信度评估
- 面向特定领域（如学术文献、法律、医疗文档）的 GraphRAG 适配

## 4. P3：VLM 副项目与评测专精（第 12-18 月）

详见 `资源/03-VLM副项目与评测/VLM副项目路线图.md`。

**项目名**：面向 VLM 的结构化负样本生成与细粒度图文对齐评测

**复用你的 Structure-CLIP 积累，升级到现代 VLM**：
1. 复现 Structure-CLIP + 读懂 ARO/NegCLIP/SugarCrepe 这条评测线。
2. **关键警示**：先读 "The Hard Positive Truth"（arXiv:2409.17958），理解难负样本的陷阱，避免重蹈覆辙。
3. 把场景图/结构化负样本方法迁移到 LLaVA / Qwen-VL / InternVL 等现代 VLM。
4. 构造细粒度图文对齐评测基准（组合性、属性绑定、关系理解）。
5. 可选延伸：VLM 幻觉评测（POPE/HallusionBench）、多模态偏好对齐（RLHF-V/POVID）。

**产出**：一个评测基准或一篇评测/方法论文，作为多模态岗位的代表作。

## 5. P4：论文产出与求职（第 18-30 月）

详见 `资源/04-论文与求职/论文与求职指南.md`。

- **论文节奏**：先冲 workshop（门槛低、快速反馈），再冲主会（ACL/EMNLP/CVPR/NeurIPS/AAAI/ICLR）。
- **简历包装**：两个代表作（主项目 + 副项目），每个用"问题-方法-结果-工程"四段式。
- **实习时间线**：第 18-24 月争取一段大厂算法实习（最好对口 RAG/Agent 或多模态团队）。
- **岗位关键词**：检索增强、知识图谱、Agent、多模态对齐、评测，匹配 JD 投递。
- **面试准备**：八股（Transformer/注意力/RAG 范式）+ 项目深挖 + 系统设计 + 手撕代码。

## 6. 每周节奏（贯穿全程）

- **投入**：20+ 小时/周。建议 60% 项目实践、25% 论文阅读、15% 基础补强。
- **每周**：精读 1-2 篇对口论文（带 GitHub 复现优先），推进主项目一个小里程碑，记一篇技术笔记。
- **每月**：完成一个可演示的子模块，复盘进度，更新"已读论文 + 已跑通项目"清单。
- **论文阅读法**：按"必读链"读（见各路线图），先读 survey 建立地图，再深挖单篇 + 复现代码。
