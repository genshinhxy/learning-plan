# RAG 系统示例知识库

## 什么是 RAG
RAG（Retrieval-Augmented Generation，检索增强生成）是一种把外部知识检索与大模型生成结合的技术。它先从知识库中检索与问题相关的文档片段，再把这些片段作为上下文交给大模型生成答案，从而减少幻觉、引入最新或私有知识。

## RAG 的三种范式
- 朴素 RAG（Naive RAG）：切块、向量化、检索 top-k、拼接生成。最简单但召回质量有限。
- 进阶 RAG（Advanced RAG）：加入查询改写、混合检索（向量+关键词）、重排序（rerank）等优化检索质量。
- 模块化 RAG（Modular RAG）：把检索、记忆、路由、融合拆成可组合的模块，灵活编排。

## GraphRAG 是什么
GraphRAG 在传统 RAG 基础上引入知识图谱。它先把文档抽取成实体和关系构成的知识图谱，检索时不仅做向量相似匹配，还能沿图结构做多跳推理，适合需要跨多个文档、多个实体关联才能回答的复杂问题。微软在 2024 年提出了代表性的 GraphRAG 方案。

## 混合检索
混合检索同时使用多种召回方式：向量检索（语义相似）、BM25（关键词匹配）、图检索（结构关联），再用倒数排名融合（RRF）或加权方式合并结果，最后用重排序模型（如 bge-reranker）精排，显著优于单一检索方式。

## 评测要点
RAG 系统的评测分硬指标和软指标。硬指标包括检索召回率、答案准确率、引用正确性、延迟和 token 成本；软指标用 LLM-as-Judge 评答案的准确性、完整性、相关性。一个生产级 RAG 必须有可复现的评测集和回归测试。
